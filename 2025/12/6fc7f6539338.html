通用公网网络端口访问不通排查流程

一、通用公网端口访问不通排查 PlantUML 流程图

二、通用公网端口访问不通文字描述总结

第一步：验证基础网络连通性（先确认链路是否通）

  • 操作：在发起访问的设备上执行 ping 公网目标IP（如 ping ``120.238.25.11）。
  • 判断与处理：
    • 若 ping 不通：说明本地到公网 IP 的网络链路有问题。先检查本地网络（如是否连对 WiFi、网线是否插好），再检查本地网关 / 路由器是否正常工作（可重启路由器），最后联系运营商确认是否存在网络故障。
    • 若 ping 通：基础网络链路正常，进入下一步端口层排查。

第二步：明确端口访问核心信息（避免信息错漏）

  • 操作：整理并记录关键信息，包括：
  1. 公网目标 IP（如业务提供的访问 IP：1.2.3.4）；
  2. 目标端口（如访问 Web 服务的 8080 端口、远程桌面的 3389 端口）；
  3. 端口对应的服务类型（如 Nginx Web 服务、MySQL 数据库服务）；
  4. 服务所在服务器的内网 IP（若服务器在局域网，如 192.168.1.100，用于后续局域网测试）。

第三步：测试端口访问状态（区分局域网 / 公网环境）

  • 操作：
    • 若在服务所在局域网（如同一公司 / 家庭网络）：执行 telnet 服务器内网IP 目标端口（如 telnet ``192.168.1.100`` 8080），或 nc -zv 服务器内网IP 目标端口（Linux/macOS）。
    • 若在公网环境（如在家访问公司服务器）：执行 telnet 公网目标IP 目标端口（如 telnet ``1.2.3.4`` 8080），或 nc -zv 公网目标IP 目标端口。
  • 判断：
    • 若端口通：说明端口访问本身正常，若业务仍有问题，需排查服务的业务逻辑（如账号密码错误、业务配置错误）。
    • 若端口不通：进入下一步，根据不通提示分类排查。

第四步：根据端口不通类型分类排查（核心步骤）

类型 1：访问提示 “超时”（请求未到达服务器）
  • 排查方向与操作：
  1. 检查端口转发规则（局域网服务器必备）：
    • 若服务所在服务器在局域网（如公司内网），需登录网关 / 路由器，进入 “端口映射”“虚拟服务器” 或 “NAT 转发” 功能，确认是否配置了 “公网端口→服务器内网 IP: 目标端口” 的规则（如公网 8080 端口转发到内网 192.168.1.100:8080），且规则状态为 “启用”。
  2. 检查服务器防火墙：
    • Linux 系统：执行 systemctl status firewalld（CentOS/RHEL）或 systemctl status ufw（Ubuntu），查看防火墙是否运行；再执行 firewall-cmd --list-ports（CentOS）或 ufw status（Ubuntu），确认目标端口是否在放行列表中，若未在则添加（如 firewall-cmd --permanent --add-port=8080/tcp 并 firewall-cmd --reload）。
    • Windows 系统：打开 “控制面板→系统和安全→Windows Defender 防火墙→高级设置”，在 “入站规则” 中查找是否有允许目标端口（如 8080）的规则，若没有则新建规则（选择 “端口”→“TCP”→输入目标端口→“允许连接”）。
  3. 确认运营商端口政策：
    • 部分运营商会屏蔽常见端口（如 80、443、8080、3389 等，80/443 需备案），可联系运营商客服，提供公网 IP 和目标端口，确认该端口是否被屏蔽，若屏蔽可申请解封或更换未被屏蔽的端口。
类型 2：访问提示 “拒绝连接”（请求到达服务器，但端口未被服务占用）
  • 排查方向与操作：
  1. 检查服务是否启动：
    • Linux 系统：执行 systemctl status 服务名（如 systemctl status nginx、systemctl status mysql），若状态为 “inactive” 则启动服务（systemctl start 服务名）。
    • Windows 系统：打开 “任务管理器→服务” 标签页，找到对应服务（如 “nginx”“MySQL”），查看 “状态” 是否为 “正在运行”，若不是则右键 “启动”。
  2. 检查服务是否监听目标端口：
    • Linux 系统：执行 ss -tulnp | grep 目标端口（如 ss -tulnp | grep 8080）或 lsof -i :目标端口，若无输出，说明服务未配置监听该端口，需修改服务配置文件（如 Nginx 的 nginx.conf 中 listen 8080;）并重启服务（systemctl restart 服务名）。
    • Windows 系统：执行 netstat -ano | findstr :目标端口（如 netstat -ano | findstr :8080），若无结果，说明服务未监听该端口，需修改服务配置（如 Tomcat 的 server.xml 中 port="8080"）并重启服务。
  3. 检查服务监听地址：
    • 若通过端口监听命令发现服务仅监听 127.0.0.1（本地回环地址），说明服务仅允许本地访问，需修改服务配置为监听 0.0.0.0（允许所有 IP 访问，包括公网）。例如 Nginx 配置 listen ``0.0.0.0:8080``;，修改后重启服务，再重新测试端口访问。

第五步：验证排查结果（确认问题是否解决）

  • 操作：再次执行 telnet 公网目标IP 目标端口 或 nc -zv 公网目标IP 目标端口，测试端口访问情况。
  • 判断与处理：
    • 若端口通：公网端口访问问题已解决，可正常使用对应服务。
    • 若端口仍不通：回到之前对应步骤，重新排查是否有遗漏（如防火墙规则添加后未重载、服务配置修改后未重启、端口转发规则配置错误等），直至端口访问正常。

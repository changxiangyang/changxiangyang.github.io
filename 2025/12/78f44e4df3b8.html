通用公网网络端口访问不通排查流程

一、通用公网端口访问不通排查 PlantUML 流程图

二、通用公网端口访问不通文字描述总结

第一步：验证基础网络连通性（先确认链路是否通）

  • 操作：在发起访问的设备上执行 ping 公网目标IP（如 ping ``120.238.25.11）。
  • 判断与处理：
    • 若 ping 不通：说明本地到公网 IP 的网络链路有问题。先检查本地网络（如是否连对 WiFi、网线是否插好），再检查本地网关 / 路由器是否正常工作（可重启路由器），最后联系运营商确认是否存在网络故障。
    • 若 ping 通：基础网络链路正常，进入下一步端口层排查。

第二步：明确端口访问核心信息（避免信息错漏）

  • 操作：整理并记录关键信息，包括：
  1. 公网目标 IP（如业务提供的访问 IP：1.2.3.4）；
  2. 目标端口（如访问 Web 服务的 8080 端口、远程桌面的 3389 端口）；
  3. 端口对应的服务类型（如 Nginx Web 服务、MySQL 数据库服务）；
  4. 服务所在服务器的内网 IP（若服务器在局域网，如 192.168.1.100，用于后续局域网测试）。

第三步：测试端口访问状态（区分局域网 / 公网环境）

  • 操作：
    • 若在服务所在局域网（如同一公司 / 家庭网络）：执行 telnet 服务器内网IP 目标端口（如 telnet ``192.168.1.100`` 8080），或 nc -zv 服务器内网IP 目标端口（Linux/macOS）。
    • 若在公网环境（如在家访问公司服务器）：执行 telnet 公网目标IP 目标端口（如 telnet ``1.2.3.4`` 8080），或 nc -zv 公网目标IP 目标端口。
  • 判断：
    • 若端口通：说明端口访问本身正常，若业务仍有问题，需排查服务的业务逻辑（如账号密码错误、业务配置错误）。
    • 若端口不通：进入下一步，根据不通提示分类排查。
